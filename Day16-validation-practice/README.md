# Day 16 – Validation Practice

Today’s focus was on reinforcing the full validation workflow by rebuilding the Members API from scratch.  
This practice round helped solidify the flow of Joi body validation, ObjectId param validation, and centralized error handling.

---

## 🔍 What Was Practiced

- Rebuilt the Members API with **Joi validation** for POST/PUT requests.
- Applied **ObjectId validation middleware** to all routes using `/:id`.
- Ensured all errors (invalid body, invalid ID, duplicate entries, not found) flow through the **centralized error handler**.
- Practiced **filtering, sorting, and pagination** again to strengthen query handling.
- Improved consistency of API responses (`success`, `data`, `message`).
